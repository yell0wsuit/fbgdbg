Take a look at the lines that are responsible for the background.

<div class="fbgdbg">
    <div class="blue-gradient"></div>
    <div class="green-gradient"></div>
    <div class="yellow-gradient"></div>
    <div class="red-gradient"></div>
    <div class="blue-gradient-bottom"></div>
    <div class="purple-gradient"></div>
</div>

In the fbgdbg class, the position property needs to be set to fixed to avoid overflowing. You can freely change the opacity property to any value you like, or remove it to let the background be shown fully--but make sure the text contrast ratio is sufficient, otherwise it will become hard to read.

Except for the yellow gradient div, every color's value is left intact. However, the background color is used with the var() function to make the code easier to read.

The yellow gradient's bottom property value is divided by 2 from the original value to have the subtle yellow color (the lower the value, the more it can be seen). Since position is set to fixed you cannot see it. While absolute can see it, there are some side effects, notably the webpage is not responsive to the screen size (overflow like I've mentioned above).
